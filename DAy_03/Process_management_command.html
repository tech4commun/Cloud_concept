Linux · Process Management

Process management commands

View, monitor, control, and kill processes on Linux — essential for managing running programs on your system or AWS EC2 instance.

1 · View processes
Snapshot of all running processes
Most commonly used — shows PID, user, CPU, memory, command
ps aux
Flags: a = all users  |  u = user-readable format  |  x = include background processes
Your processes only
Processes by current user
ps -u $USER
Filter by name
Find a specific process
ps aux | grep nginx
2 · Real-time monitoring
top — live process viewer
Refreshes every 3 seconds — shows CPU, memory usage in real time
top
Inside top: press q to quit  |  k to kill  |  M sort by memory  |  P sort by CPU
htop — better interactive viewer
Color UI, mouse support, easier to use than top (install first)
sudo apt install htop -y
htop
Tip: On AWS EC2 Amazon Linux — sudo yum install htop
3 · Find process ID (PID)
pgrep
Returns PID by process name
pgrep nginx
Example output: 1234
pidof
Returns PID of exact program name
pidof nginx
Difference: pgrep = partial match, pidof = exact name.
4 · Kill / stop processes
kill — stop by PID
Sends a signal to a process using its PID
kill 1234
kill -9 1234
Note: Default is SIGTERM (graceful). -9 is SIGKILL (force — no cleanup).
killall
Kill all processes by name
killall nginx
pkill
Kill by partial name match
pkill -f nginx

kill -9 is a last resort — the process cannot save data or clean up. Always try kill <PID> first and only use -9 if it doesn't respond.

5 · Common kill signals
kill -1
SIGHUP
Reload config without restart
kill -2
SIGINT
Same as Ctrl+C interrupt
kill -9
SIGKILL
Force kill — no cleanup
kill -15
SIGTERM
Graceful stop (default)
kill -19
SIGSTOP
Pause/suspend process
kill -18
SIGCONT
Resume a stopped process
6 · Background and foreground
Run in background
Append & to any command
python3 app.py &
Output: [1] 5678 — job number and PID
List background jobs
See all jobs in current session
jobs
Bring to foreground
Resume background job in terminal
fg %1
Note: %1 = job number from jobs list
Suspend to background
Pause a running foreground process
Ctrl + Z
Then run bg to resume in background.
7 · Keep process alive after logout
nohup — no hangup
Keeps process running even after you log out of the SSH session
nohup python3 app.py &
Output saved to: nohup.out by default in current directory.

On AWS EC2, use nohup when running scripts over SSH so they don't stop when your connection drops. For long-term services, use systemd or screen/tmux instead.

Quick reference cheatsheet
ps aux               # all running processes
ps aux | grep nginx    # find specific process
top                   # live monitor (q to quit)
htop                  # better live monitor
pgrep nginx           # find PID by name
kill 1234             # graceful stop
kill -9 1234          # force kill
killall nginx         # kill all by name
python3 app.py &      # run in background
jobs                  # list background jobs
fg %1                 # bring job to foreground
nohup python3 app.py & # keep alive after logout
