Linux · Networking Commands

Networking commands

Check IPs, test connectivity, monitor ports, transfer files, and debug network issues — all essential for Linux and AWS EC2 work.

1 · Check IP address
ip addr — show all IP addresses
Modern command — shows all network interfaces and their IPs
ip addr
ip addr show eth0
Look for: inet = IPv4 address  |  inet6 = IPv6 address
ifconfig
Older command, still common
ifconfig
ifconfig eth0
Install: sudo apt install net-tools
Public IP
Your external/public IP address
curl ifconfig.me
curl icanhazip.com
AWS EC2: Useful to find your instance's public IP.
2 · Test connectivity
ping — test if host is reachable
Sends packets and measures response time
ping google.com
ping -c 4 google.com
Flag: -c 4 = send only 4 packets then stop. Without it, runs forever — stop with Ctrl+C
traceroute — trace path to host
Shows every hop/router the packet passes through
traceroute google.com
tracepath google.com
Install: sudo apt install traceroute  |  tracepath is built-in alternative

On AWS EC2, if ping fails to reach your instance, check your Security Group inbound rules — ICMP traffic must be allowed.

3 · DNS lookup
nslookup
Query DNS for a domain
nslookup google.com
Shows: IP address and DNS server used
dig
Detailed DNS information
dig google.com
dig google.com A
Install: sudo apt install dnsutils
host — quick DNS resolve
Simplest way to resolve a domain to IP
host google.com
Output: google.com has address 142.250.x.x
4 · Port and socket info
ss — socket statistics
Modern replacement for netstat — shows open ports and connections
ss -tuln
ss -tulnp
Flags: t = TCP  |  u = UDP  |  l = listening  |  n = numeric  |  p = show process
netstat
Older but widely used
netstat -tuln
netstat -tulnp
Install: sudo apt install net-tools
Check specific port
Is port 80 open and listening?
ss -tuln | grep :80

On AWS EC2, even if a port is open on Linux, traffic won't reach it unless the Security Group has an inbound rule allowing that port.

5 · Download and transfer files
curl — transfer data from URL
Download files, call APIs, test HTTP endpoints
curl https://example.com
curl -O https://example.com/file.zip
curl -I https://example.com
Flags: -O = save file  |  -I = headers only  |  -X POST = HTTP method
wget — download files
Best for large files — supports resume
wget https://example.com/file.zip
wget -c https://example.com/file.zip
Flag: -c = resume interrupted download
scp — secure copy over SSH
Transfer files to/from a remote server or EC2 instance
scp file.txt user@server:/home/user/
scp -i key.pem file.txt ubuntu@ec2-ip:/home/ubuntu/
AWS EC2: Use -i key.pem flag with your key pair to authenticate.
6 · SSH — connect to remote server
ssh — secure shell connection
Log into remote Linux machines securely over the network
ssh user@hostname
ssh ubuntu@192.168.1.10
ssh -i key.pem ubuntu@ec2-public-ip
AWS EC2: Default user is ubuntu (Ubuntu) or ec2-user (Amazon Linux)
SSH with port
Connect on a custom port
ssh -p 2222 user@host
SSH with verbose
Debug connection issues
ssh -v user@host
7 · Network monitoring
nload
Real-time bandwidth monitor
sudo apt install nload
nload
iftop
Live per-connection bandwidth
sudo apt install iftop
sudo iftop
netstat -s — network statistics summary
Total packets sent, received, errors by protocol
netstat -s
8 · Routing table
ip route
View routing table (modern)
ip route
ip route show
route
View routing table (older)
route -n
Flag: -n = show numeric IPs (faster)

On AWS EC2, the routing table is managed by the VPC. Use ip route to see the default gateway — usually the first IP in your subnet (e.g. 172.31.0.1).

Quick reference cheatsheet
ip addr                  # show IP addresses
curl ifconfig.me          # public IP
ping -c 4 google.com      # test connectivity
traceroute google.com     # trace network path
nslookup google.com       # DNS lookup
dig google.com            # detailed DNS info
ss -tuln                  # open ports
ss -tuln | grep :80       # check specific port
curl -O <url>             # download file
wget <url>                # download file
scp -i key.pem f user@ip:/ # copy to EC2
ssh -i key.pem ubuntu@ip  # connect to EC2
ip route                  # routing table
