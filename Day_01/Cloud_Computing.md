# Cloud Computing
>delivery of on-demand computing services, including storage, databases, and applications, over the internet
>providing scalability and flexibility without the need for any physical servers or systems.
>Cloud Computing does not imply that there is only one cloud. The phrase "Cloud" refers to the Internet, a network. Furthermore, not all forms of remote computing are Cloud Computing. 

# Cloud
>is a set of services, networks, hardware, storage, and interfaces that enable the delivery of computing as a service. 
>These resources include applications, processing, storage, networking, development, and deployment platforms, and business processes.

# Cloud Technology
>The cloud is an environment in which multiple servers, networks, hardware, storage, and interfaces are located.
>They help in the delivery of computing as a service.
>It broadly has three users:
1.end user -uses the service provided by the cloud.
2.business and management user --responsible for the data and services provided by the user.
3.cloud service provider ---takes care of the IT assets of the cloud.

# cloud service models
1.Infrastructure-as-a-Service: IaaS--
>offers virtualized computing resources(servers, storage, and networking)over the internet.
>giving users control over their own operating systems and applications without managing physical hardware.

>Examples: Amazon Web Services (AWS EC2), Microsoft Azure Virtual Machines, Google Compute Engine (GCE), and DigitalOcean.

2. Platform-as-a-Service (PaaS)
>Provides a cloud-based environment with built-in tools for developing, testing, hosting, and deploying applications.
>freeing developers from managing the underlying infrastructure.

>Examples: Heroku, AWS Elastic Beanstalk, Google App Engine, and Microsoft Azure App Services.

3. Software-as-a-Service (SaaS)
>SaaS delivers complete software applications over the internet through subscriptions.
>removing the need to download or install programs locally.

>Examples: Google Workspace (Gmail, Docs), Microsoft 365, Salesforce, Netflix, and Slack.

4. Function-as-a-Service (FaaS) / Serverless
>Allows developers to deploy individual pieces of code or functionsthat trigger in response to specific events
>charging only for the exact resources consumed during execution.

Examples: AWS Lambda, Google Cloud Functions, and Azure Functions.
